HACKING EXTENSION SYSTEM
目的: 目押しの認証範囲をコード編集で広げる
ゲームに戻る

まず何をする？

  1. 「テンプレートを読み込む」を押す
  2. `HackScript` で `probeBudget` / `stabilityBias` / `entropyGate` を調整する
  3. 必要ならストアでヒントやモジュールを購入する
  4. 「検証して保存」でゲームへ戻る
初心者向けの目安
`probeBudget`: 1〜5（高いほど推定が安定）
`stabilityBias`: 0〜100（高いほど判定幅が広がりやすい）
`entropyGate`: 0〜100（sweet spot への寄り方に影響）
HackScript の書き方
関数型スタイルと `class` スタイルの両方をパースできます。
chapter chapter1 {
  probeBudget(3);
  stabilityBias(72);
  entropyGate(48);
  focusMode("safe");
}
class Chapter2 extends ExploitProfile {
  probeBudget = 4;
  stabilityBias = 58;
  entropyGate = 61;
  focusMode = "balanced";
}
編集ルール
`sweetLeft` / `sweetWidth` の直接指定は禁止
1ボタンでの sweet spot 自動特定は廃止
`focusMode`: `safe`(安定) / `balanced`(標準) / `greedy`(高配当寄り)
コメントは `//`, `#`, `/* ... */` が使えます
攻略は「コード調整 + ストア購入」で段階解放

ハッキングストア

所持ポイント: 0pt
購入済み機能
ここで保存した設定は、本編の目押し判定にすぐ反映されます。
